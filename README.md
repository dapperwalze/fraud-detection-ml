# Fraud Detection ML System

An end-to-end machine learning system that detects fraudulent credit card transactions, trained on 284,807 real transactions and deployed as a live REST API.

**Live API:** https://fraud-detection-ml-is5l.onrender.com/docs

---

## Results

| Model | Precision | Recall | F1 | ROC AUC | Train Time |
|-------|-----------|--------|----|---------|------------|
| Logistic Regression | 83% | 64% | 0.72 | 0.956 | 1.74s |
| Random Forest | 94% | 82% | 0.87 | 0.953 | 25.65s |
| RF + SMOTE | 87% | 83% | 0.85 | 0.973 | 45.33s |
| **XGBoost (winner)** | **88%** | **85%** | **0.86** | **0.969** | **0.92s** |

XGBoost outperformed all models on recall and precision simultaneously, trained 50× faster than RF + SMOTE, and required no synthetic data pipeline.

## Dataset

Credit Card Fraud Detection dataset from Kaggle (ULB Machine Learning Group).
284,807 transactions — 0.173% fraud rate.
Download: https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

Place `creditcard.csv` in the project root before running the notebook.

## Setup

```bash
# Create and activate environment
conda create -n fraud-ml python=3.11 -y
conda activate fraud-ml

# Install dependencies
pip install -r requirements.txt

# Run the notebook
jupyter notebook fraud_detection.ipynb

# Start the API locally
uvicorn api:app --reload
```

Visit `http://127.0.0.1:8000/docs` for interactive API documentation.

## API Usage

**POST /predict** — accepts 29 transaction features, returns fraud assessment.

Sample legitimate transaction response:
```json
{
  "is_fraud": false,
  "fraud_probability": 0.0,
  "risk_level": "LOW"
}
```

Sample fraud transaction response:
```json
{
  "is_fraud": true,
  "fraud_probability": 1.0,
  "risk_level": "HIGH"
}
```

Risk levels: `LOW` (< 30% probability) · `MEDIUM` (30–70%) · `HIGH` (> 70%)

## Key Learnings

- Accuracy is misleading for imbalanced datasets — Recall is the metric that matters
- XGBoost's `scale_pos_weight` replaces the entire SMOTE pipeline with one parameter
- Data preparation (3 days) takes far longer than model training (< 1 second)
- MLflow experiment tracking makes results reproducible and comparable across runs
- The full ML pipeline: data → explore → prepare → train → evaluate → deploy → monitor
