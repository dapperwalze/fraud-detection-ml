# Fraud Detection ML System

An end-to-end machine learning system that detects fraudulent credit card transactions, trained on 284,807 real transactions and deployed as a REST API.

## Results

| Model | Precision | Recall | F1 | ROC AUC |
|-------|-----------|--------|----|---------|
| Logistic Regression | 83% | 64% | 0.72 | 0.956 |
| Random Forest | 94% | 82% | 0.87 | 0.953 |
| **RF + SMOTE (winner)** | **87%** | **83%** | **0.85** | **0.973** |

## Dataset
Credit Card Fraud Detection dataset from Kaggle (ULB Machine Learning Group).
284,807 transactions — 0.173% fraud rate.
Download from: https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

## Stack
- Python 3.11
- pandas, numpy, matplotlib, seaborn
- scikit-learn, imbalanced-learn
- FastAPI, uvicorn, joblib

## Setup
```bash
# Create environment
conda create -n fraud-ml python=3.11 -y
conda activate fraud-ml
pip install pandas numpy matplotlib seaborn scikit-learn jupyter imbalanced-learn fastapi uvicorn joblib

# Run the notebook
jupyter notebook fraud_detection.ipynb

# Start the API
uvicorn api:app --reload
```

## API Usage
Once running, visit http://127.0.0.1:8000/docs for interactive documentation.

POST /predict with transaction features returns:
```json
{
  "is_fraud": false,
  "fraud_probability": 0.0,
  "risk_level": "LOW"
}
```

## Key Learnings
- Accuracy is misleading for imbalanced datasets — Recall matters more
- SMOTE balances training data by creating synthetic fraud examples
- Random Forest outperformed Gradient Boosting on this dataset
- The full ML pipeline: data → explore → prepare → train → evaluate → deploy
